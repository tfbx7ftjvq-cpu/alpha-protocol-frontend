# Fair Launch Tokenomics Correction / Allocation Model Superseded

## 1. Purpose

This document supersedes the previous ALPHA token allocation review.

The earlier `35/20/15/15/10/5` token bucket model is no longer the recommended ALPHA launch model. Alpha Protocol is now documented as a Fair Launch token model.

## 2. Superseded Previous Allocation Model

The previous allocation model is superseded:

- DAO Treasury / Protocol Reserve `35%`.
- Community / Ecosystem Incentives `20%`.
- Liquidity / Launch Pool `15%`.
- Staking Rewards Reserve `15%`.
- Builders / Contributors `10%`.
- Team / Founding Contributors `5%`.

The previous Option A / Option B / Option C comparison is also superseded.

These concepts must not be presented as active launch recommendations.

## 3. Correct Fair Launch Model

Alpha Protocol will not use initial token allocation buckets.

Correct model:

- Launch model: Fair Launch.
- Project-side reserved token allocation: none.
- Founder / project-side token bucket: none.
- Team preallocation: none.
- VC allocation: none.
- DAO treasury token allocation at launch: none.
- Staking token reserve at launch: none.
- Builders / contributors token allocation at launch: none.
- Market participants acquire ALPHA through the open market after launch.

There is no hidden project-side token reserve.

## 4. Founder / Project-Side Position

Founder / project side does not receive a preallocated ALPHA bucket.

Any future contributor compensation should come from:

- Protocol revenue.
- DAO-approved contributor budget.
- Transparent future governance.
- Separately disclosed arrangements.

It should not come from a hidden initial token allocation.

## 5. Revenue Split, Not Token Allocation

The only standing split is the protocol revenue split after revenue enters Treasury.

Correct model:

Protocol Revenue / Project Revenue
-> Treasury
-> 50% Relief Pool
-> 20% Buyback / Burn
-> 20% DAO payroll / builders / contributors
-> 10% Staking Rewards

This is not ALPHA token supply allocation.

## 6. Why the Allocation Model Was Replaced

Fair Launch reduces insider allocation complexity by removing:

- Team token bucket.
- VC token bucket.
- Project-side reserve.
- DAO treasury launch allocation.
- Staking token reserve.
- Builders token allocation.

Fair Launch is easier to explain publicly because market participants access ALPHA through the open market after launch.

Fair Launch does not mean risk-free.

## 7. Staking Rewards Correction

Staking rewards come from the `10%` Treasury revenue split.

They do not come from a preallocated ALPHA staking reserve.

Rewards depend on real protocol revenue, pool balance, protocol rules, and future governance.

No fixed APY and no guaranteed yield should be stated or implied.

## 8. Buyback / Burn Correction

Buyback / burn comes from `20%` of protocol revenue.

It is not:

- Guaranteed buyback.
- Price floor support.
- Price protection.
- Guaranteed liquidity support.
- A promise of token price appreciation.

## 9. DAO / Builders / Contributors Correction

`20%` of protocol revenue may support DAO payroll, builders, contributors, maintenance, audits, and operations.

This is revenue-based, not a preallocated token bucket.

Future spending should be transparent and governance-reviewed.

## 10. Launch Readiness Still Required

Fair Launch still requires:

- Launch platform.
- Initial liquidity setup.
- Pairing asset decision, such as SOL or USDC.
- Mint authority policy.
- Freeze authority policy.
- Liquidity custody / LP handling.
- Anti-scam communication.
- Risk / legal wording review.
- Final Go/No-Go review.

## 11. User Decisions Remaining

The user still needs to decide:

- Whether total supply is `1,000,000,000 ALPHA`.
- Launch platform.
- Initial liquidity setup.
- Pairing asset, such as SOL or USDC.
- Mint authority policy.
- Freeze authority policy.
- Official launch communication.
- Anti-scam warnings.
- Timing of community preheating.
- Timing of token launch.

## 12. Decisions No Longer Required

The user no longer needs to decide:

- Team allocation.
- Team vesting.
- Builders token vesting.
- Staking token reserve.
- DAO treasury token allocation.
- Airdrop bucket.
- Allocation Option A / B / C.

## 13. Current Conclusion

- Previous allocation model: superseded.
- Correct model: Fair Launch.
- Token launch: still NO-GO.
- Mainnet production: still NO-GO.
- Community preheating: may explain Fair Launch with warnings.
