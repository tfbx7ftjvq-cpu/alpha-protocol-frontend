## 4. Treasury Revenue Split

`50 / 20 / 20 / 10` is the split rule for future protocol / project revenue after it enters Treasury. It is not initial ALPHA token allocation.

Protocol Revenue / Project Revenue
-> Treasury
-> 50% Relief Pool
-> 20% Buyback / Burn
-> 20% DAO payroll / builders / contributors
-> 10% Staking Rewards

Additional constraints:

- Staking rewards come from treasury revenue split, not an initial ALPHA staking reserve.
- Buyback / burn is not guaranteed buyback.
- Buyback / burn is not price floor support.
- Relief Pool does not mean automatic payout.
